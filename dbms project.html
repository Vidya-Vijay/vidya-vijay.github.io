Chemical Production Plant Management System: SQL Database for Operations

Capstone project simulating the design of a SQL-based database to manage chemical plant operations, showcasing relational modeling, SQL expertise, and operational reporting skills.

SQL ERD

Project Summary

Built a simulated SQL-based relational database system to manage operations in a chemical production plant, supporting roles like plant managers, QA teams, and inventory staff. The system automates production batch scheduling, raw material tracking, order processing, and payment management, demonstrating end-to-end database design and optimization skills.

Business Context

Chemical production plants rely on accurate inventory, production, and order data. Without a structured database, managers face inefficiencies, errors in scheduling, and delays in customer order fulfillment. This simulated project demonstrates the ability to design systems that optimize operations and ensure data integrity, even in complex multi-entity environments.

Problem Statement

Design a database system that ensures data integrity, reduces redundancy, and provides managers and staff with reliable tools to monitor production, inventory, and customer orders efficiently.

Tools & Methods

  • Tools: MySQL, phpMyAdmin
  • Techniques: Relational Modeling, Normalization (2NF), Entity Relationship Diagram (ERD), DDL & DML, SQL Query Optimization, JOINs, GROUP BY, Subqueries, Aggregations

Key Features/Highlights

  • Normalized Schema Design: Created 10+ interrelated tables in 2NF with clear functional dependencies to reduce redundancy and ensure integrity.
  • Entity Relationship Diagram: Mapped entities like Raw_Material, Supplier, Product, Order, Customer, and Payment using Chen’s notation.
  • Relational Mapping: Used bridge tables (Product_Material, Order_Details) to model M:N relationships.
  • DDL & Implementation: Built schema using MySQL and implemented all relationships with appropriate data types, constraints, and indexing for query efficiency.
  • SQL Queries: Developed advanced queries using JOIN, GROUP BY, subqueries, and aggregation to generate reports on order status, raw material usage, and customer payments.

Business Impact

While simulated, the project demonstrates the potential impact of a well-designed database system:

  • Efficient tracking of production and inventory
  • Streamlined order processing and payment management
  • Reduced errors and improved operational decision-making
