Chemical Production Plant Management System

A SQL-based database system for managing chemical production plant operations

Course: COMP 230 – Databases and Database Management Systems

Project Overview

This project involved designing and implementing a relational database system to manage operations in a chemical production plant. The system streamlined production batch scheduling, raw material inventory tracking, customer order processing, and payment management. Built to support roles like plant managers, QA, and inventory teams, this project showcased end-to-end DBMS skills.

Tools & Techniques

  • MySQL Database
  • phpMyAdmin for database interface
  • SQL: Data Definition Language (DDL), Data Manipulation Language (DML)
  • Entity Relationship Diagram (ERD) and Normalization

Key Features

  • Normalized Schema Design: Created 10+ interrelated tables in 2NF with clear functional dependencies to reduce redundancy and ensure integrity.
  • Entity Relationship Diagram: Mapped entities like Raw_Material, Supplier, Product, Order, Customer, and Payment using Chen’s notation.
  • Relational Mapping: Used bridge tables (Product_Material, Order_Details) to model M:N relationships.
  • DDL & Implementation: Built schema using MySQL and implemented all relationships with appropriate data types, constraints, and indexing for query efficiency.
  • SQL Queries: Developed advanced queries using JOIN, GROUP BY, subqueries, and aggregation to generate reports on order status, raw material usage, and customer payments.
